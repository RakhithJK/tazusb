TazUSB Manual

Nom

TazUSB - SliTaz LiveUSB utility

SYNTAXE

tazusb [command] [compression|device|file]

DESCRIPTION

TazUSB est un utilitaires conçu pour installer SliTaz sur un média USB. A la différence d'une installation classique, le système de fichier demeure dans un fichier compressé "rootgs.gz". Le système de fichier est entièrement chargé en mémoire au démarrage. Ceci augmente la réactivité, protège le système de fichier contre une corruption accidentelle et réduit la lecture/écrite sur le média USB. Cet utilitaire peut aussi ré-écrire le système de fichier racine avec les changements que vous avez fait depuis le démarrage, permettant de profiter des avantages d'une installation sur disque dur.

TazUSB supporte les lecteurs formatés en FAT32/EXT3 utilisant respectivement SYSLINUX et EXTLINUX. "/home" est monté au démarrage en utilisant le UUID de votre lecteur flash. Contrairement à un nom de périphérique, l'UUID a l'avantage de ne jamais changer d'une machine à une autre.

COMMANDES

usage

La commande 'usage' affiche un bref sommaire de toutes les commandes disponibles. 

 # tazusb usage

writefs

La commande 'writefs' compresse le système de fichier actuellement en mémoire dans un fichier "rootfs.gz". Si votre lecteur flash est monté en tant que /home (cela devrait être le cas), le nouveau système de fichier sera copié dessus pour vous, sinon il sera placé à la racine du système. Votre précédent système de fichier sera renommé "previous.gz" et sera accessible au démarrage en entrant "previous" à l'écran "boot:". Tous les système de fichiers précédents sont renommés en rootfs.gf.unixtimestamp. Ils ne sont pas supprimés automatiquement, donc vous devrez les retirer périodiquement pour conserver de l'espace-disque.

Les options de compression du système de fichier sont : lzma, gzip, none (aucune). Ne pas utiliser de compression est très rapide (moins de 5 secondes) et utile lors d'expérimentations avec beaucoup de changements. En comparaison, utiliser lzma ou gzip prend quelques minutes mais réduit énormément la taille du fichier. Ceci est recommandé lors de la sauvegarde de changements permanents sur le système de fichier. 

 # tazusb writefs compression
Exemple: 
 # tazusb writefs lzma

format

La commande 'format' est utilisée pour formater un périphérique afin de l'utiliser comme périphérique LiveUSB. Actuellement, elle permet le formatage en EXT2, EXT3 et FAT32. 

 # tazusb format /dev/name
Exemple: 
 # tazusb format /dev/sda1

gen-liveusb

'gen-liveusb' installe une nouvelle MBR, rend votre partition bootable (démarrable) et installe syslinux/extlinux selon le système de fichier détecté. Ensuite, il copie le noyau et le système de fichier depuis le lecteur CD-ROM, et les place dans le média USB cible. Le résultat est une copie de SliTaz bootable sur média USB. 

 # tazusb gen-liveusb /dev/name
Exemple: 
 # tazusb gen-liveusb /dev/sda1

gen-swap

La commande 'gen-swap' (re)créé un fichier swap (extension de mémoire vive sur un disque) et le place dans le répertoire /home pour être utilisable à chaque démarrage. Ceci est utile pour l'utilisation de vielles machine avec peu de mémoire. 

 # tazusb gen-swap

gen-iso2usb

Cette commande est similaire à gen-liveusb, mais utiliser une image ISO téléchargée à la place du lecteur CD-ROM. 

 # tazusb gen-iso2usb /path/to/iso
Exemple: 
 # tazusb gen-iso2usb /home/tux/slitaz.iso

clean

'clean' retire les anciens systèmes de fichiers rootfs.gz.unixtimestamp pour conserver de l'espace-disque (voir writefs). 

 # tazusb clean
